1. Module Declaration

The outer shell of any hardware block. Lists all inputs and outputs.

module mac_unit (
    input  wire                clk,            // 1-bit clock
    input  wire                load_weight,    // 1-bit control signal
    input  wire signed [7:0]   weight_in,      // 8-bit signed input
    input  wire signed [7:0]   activation_in,  // 8-bit signed input
    input  wire signed [31:0]  psum_in,        // 32-bit signed input
    output wire signed [31:0]  psum_out,       // 32-bit signed output
    output wire signed [7:0]   activation_out  // 8-bit passthrough
);

    // internals go here

endmodule
Note: No comma after the last port. No semicolon after the port list closing parenthesis — the semicolon goes after ).

2. Wire vs Reg

wire — a connection. No storage. Value is always driven by something.

reg — can hold a value. Used inside always blocks. Becomes a flip-flop when used with a clock.

// Wire: just a connection
wire [15:0] product;
assign product = a * b;
// Reg: stores a value
reg [7:0] weight_reg;
// assigned inside always block
Rule: If you drive a signal with assign, declare it as wire. If you drive it inside an always block, declare it as reg. Mixing these up = compiler error.

3. Assign (Combinational Logic)

Creates a permanent connection. Output updates instantly when inputs change. No clock.

// These run ALL THE TIME, in parallel
assign sum         = a + b;
assign is_negative = (value[31] == 1'b1);
assign data_out    = data_in;  // passthrough — just a wire

4. Always Block — Combinational

Alternative to assign for more complex combinational logic. Uses blocking =.

always @(*) begin
    if (select)
        out = a;
    else
        out = b;
end
@(*) means "recalculate whenever ANY input changes." Use blocking = here.

5. Always Block — Clocked (Sequential)

Updates on clock edge only. Creates flip-flops. Uses non-blocking <=.

always @(posedge clk) begin
    if (load)
        weight_reg <= weight_in;   // store new value on clock edge
    // else: weight_reg keeps its old value automatically
end
Critical: Use <= (non-blocking) in clocked blocks. Use = (blocking) in combinational blocks. Mixing them up causes simulation bugs.

6. Signed Numbers

By default, Verilog treats everything as unsigned. You MUST declare signed for INT8 math.

wire signed [7:0] a;       // range: -128 to +127
wire        [7:0] b;       // range: 0 to 255 (WRONG for INT8!)

// $signed() forces signed interpretation
assign product = $signed(a) * $signed(b);
Bug #3: Forgetting signed turns -3 into 253. Your entire model outputs garbage. Always declare INT8 signals as signed.

8. Sign Extension

Expanding a smaller signed number to a larger width. Critical for MAC unit.

wire signed [15:0] product;     // 16-bit from multiplier
wire signed [31:0] product_ext; // 32-bit for adder

// Method 1: If both declared as signed, Verilog does it automatically
assign product_ext = product;   // auto sign-extends if signed

// Method 2: Manual sign extension (copies sign bit 16 times)
assign product_ext = {{16{product[15]}}, product};
//                    ^^^^^^^^^^^^^^^^  ^^^^^^^^
//                    16 copies of       original
//                    the sign bit       16 bits
Tip: If everything is declared signed, Verilog handles sign extension automatically on assignment. But it's good to know the manual way.

10. Ternary (Multiplexer)

Selects between two values based on a condition. Creates a MUX in hardware.

// If condition is true, use a. Otherwise, use b.
assign out = (condition) ? a : b;

// Example: clear the accumulator or keep adding
assign next_acc = (clear) ? 32'd0 : (acc + product);

11. Module Instantiation

Using a module inside another module. Like calling a function, but it creates physical hardware.

// Create one instance of mac_unit called "mac0"
mac_unit mac0 (
    .clk(clk),
    .load_weight(load_signal),
    .weight_in(w_data),
    .activation_in(act_data),
    .psum_in(32'd0),           // leftmost MAC, no previous sum
    .psum_out(result),
    .activation_out(act_pass)
);

// Create another — two separate circuits, both running in parallel
mac_unit mac1 (
    .clk(clk),
    .psum_in(result),          // chained! mac0's output → mac1's input
    // ... etc
);

12. State Machines (FSM)

How you make things happen in sequence in hardware.

// Define states
localparam IDLE    = 2'd0;
localparam LOAD    = 2'd1;
localparam COMPUTE = 2'd2;
localparam DONE    = 2'd3;

reg [1:0] state;

always @(posedge clk) begin
    case (state)
        IDLE:    state <= LOAD;
        LOAD:    if (load_done)    state <= COMPUTE;
        COMPUTE: if (compute_done) state <= DONE;
        DONE:    state <= IDLE;
    endcase
end

13. Parameters

Constants that make modules reusable with different sizes.

module mac_unit #(
    parameter DATA_WIDTH = 8,
    parameter ACC_WIDTH  = 32
)(
    input wire signed [DATA_WIDTH-1:0] weight_in,
    input wire signed [ACC_WIDTH-1:0]  psum_in,
    // ...
);

15. Common Gotchas

1. = vs <=
Use = in combinational (assign, always @(*)).
Use <= in clocked (always @(posedge clk)).
Mix them up → simulation behaves differently from real hardware.
2. Forgetting signed
wire [7:0] x = 8'hFF; → x is 255 (unsigned)
wire signed [7:0] x = 8'hFF; → x is -1 (signed)
For INT8 neural network math, EVERYTHING must be signed.
3. Width mismatches
Adding an 8-bit and 32-bit number without sign extension → upper bits are garbage.
Always match widths explicitly or let signed handle it.
4. No comma after last port
The last signal in a module port list has NO comma. Compiler error if you add one.
5. Undriven signals
If you declare a reg but never assign it in an always block, it holds x (unknown) in simulation. Make sure every reg gets assigned in all paths.
